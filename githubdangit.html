Github dan Git

Kali ini saya akan sharing tentang 2 hal tersebut. Mungkin juga udah banyaknya yah tentunya, tentang pembahasan Github dan Git ini. Saya sharing sepengetahuan dan saya juga masih belajar :), Ok langsung saja.

Github ini seperti tempat penyimpanan dan pusat kolaborasi untuk pengembangan project. Nah dalam github ini terdapat banyak repository git. Untuk dapat menggunakan, teman-teman dapat mengunjungi https://github.com, selain Github teman-teman juga bisa menggunakan seperti Bitbucket, atau Gitlab (ini yang saya tahu ya, kalau temen-temen ada informasi lagi monggo silahkan bisa dishare juga)

Git ini termasuk Version Control System. Version Control System sendiri termasuk tools perangkat lunak yang membantu software team dalam mengelola perubahan source code dari waktu ke waktu.

Dasar-dasar Git
  • Git hanya menyimpan snapshot. Jadi singkatnya jika berkas-berkas itu tidak berubah, Git tidak menyimpan berkasnya lagi, tapi hanya menautkan pada berkas yang sama persis sebelumnya yang telah tersimpan.
  • Git memiliki 3 status: `comitted` (data sudah tersimpan di database), `staged` (menandai pada file yang mengalami perubahan), `modified` (perubahan file belum ditandai)
  • Hampir setiap operasi Git di local. Jadi Git tidak perlu terhubung ke server untuk mendapatkan history dan menampilkan, cukup membaca langsung dari database lokal.
